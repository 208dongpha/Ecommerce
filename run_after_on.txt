1. Chạy demo hằng ngày (khuyên dùng)

Khởi động toàn bộ hệ thống:

docker compose up -d

Kiểm tra:

docker ps

Chạy:

Producer
cd producer
python producer.py
Bronze
docker exec -it spark-master bash
SPARK_SUBMIT_OPTS="-Divy.cache.dir=/tmp/.ivy2/cache -Divy.home=/tmp/.ivy2" \
/opt/spark/bin/spark-submit \
--master spark://spark-master:7077 \
--total-executor-cores 4 \
--packages org.apache.spark:spark-sql-kafka-0-10_2.12:3.5.1,org.apache.hadoop:hadoop-aws:3.3.4 \
/opt/spark-apps/streaming_job.py
Silver

Mở terminal mới:

docker exec -it spark-master bash
SPARK_SUBMIT_OPTS="-Divy.cache.dir=/tmp/.ivy2/cache -Divy.home=/tmp/.ivy2" \
/opt/spark/bin/spark-submit \
--master spark://spark-master:7077 \
--total-executor-cores 4 \
--packages org.apache.hadoop:hadoop-aws:3.3.4 \
/opt/spark-apps/silver_job.py

Sau đó để chạy liên tục.

2. Muốn refresh Dashboard

Khi đã có dữ liệu mới:

Chạy Gold
docker exec -it spark-master bash
SPARK_SUBMIT_OPTS="-Divy.cache.dir=/tmp/.ivy2/cache -Divy.home=/tmp/.ivy2" \
/opt/spark/bin/spark-submit \
--master spark://spark-master:7077 \
--packages org.apache.hadoop:hadoop-aws:3.3.4 \
/opt/spark-apps/batch_job.py
Load Gold → PostgreSQL
SPARK_SUBMIT_OPTS="-Divy.cache.dir=/tmp/.ivy2/cache -Divy.home=/tmp/.ivy2" \
/opt/spark/bin/spark-submit \
--master spark://spark-master:7077 \
--packages org.apache.hadoop:hadoop-aws:3.3.4,org.postgresql:postgresql:42.7.3 \
/opt/spark-apps/load_gold_to_postgres.py

Sau đó vào Superset:

Dashboard → Refresh

là thấy số liệu mới.
